# 用户头像获取功能说明

## 一、功能概述

系统根据用户提供的邮箱地址，智能选择最优的头像获取方式，确保在各种场景下都能正确展示用户头像。整体采用 **三级兜底策略**：

```
邮箱输入
  │
  ├─► 判断是否为 QQ 邮箱 ──► 是 ──► 获取 QQ 官方头像 (qlogo.cn)
  │                                    │
  │                                    └──► 失败？ ──► 前端兜底 ui-avatars
  │
  └─► 否（非 QQ 邮箱） ──► 获取 Gravatar 头像 (gravatar.com)
                                     │
                                     └──► 失败？ ──► 前端兜底 ui-avatars
```

## 三、QQ 头像获取详解

### 3.1 核心原理

QQ 邮箱的域名部分特征非常明确（仅支持 `@qq.com` 和 `@vip.qq.com`），且用户名是纯数字 QQ 号。因此我们可以从邮箱中提取 QQ 号，直接调用 QQ 官方的头像接口 `q1.qlogo.cn` 来获取该用户的真实头像（前提是该 QQ 号是真实存在的）。

### 3.2 邮箱匹配正则

```regex
^(\d+)@(?:vip\.)?qq\.com$
```

| 正则片段 | 含义 |
|---|---|
| `^(\d+)` | 匹配开头的纯数字用户名（捕获组1 = QQ号） |
| `@` | 匹配 @ 符 |
| `(?:vip\.)?` | 可选匹配 `vip.` 前缀（非捕获组） |
| `qq\.com` | 匹配 `qq.com` 域名 |
| `$` | 字符串结尾（防止匹配 `qqq.com` 等） |

### 3.3 QQ 头像接口格式

```
https://q1.qlogo.cn/g?b=qq&nk={QQ号}&s={尺寸}
```

| 参数 | 说明 |
|---|---|
| `b=qq` | 固定值，指定头像类型为 QQ 头像 |
| `nk` | QQ 号（字符串或数字均可） |
| `s` | 图片尺寸（**必须为 qlogo.cn 支持的有效值**，见下文） |

### 3.4 ⚠️ 尺寸参数注意事项（踩坑记录）

**qlogo.cn 接口不支持任意尺寸**，仅对特定尺寸返回 HTTP 200，其他尺寸一律返回 **HTTP 400 Bad Request**！

经实测，以下尺寸是 qlogo.cn **支持**的：

| 尺寸值 | 是否支持 | 备注 |
|---|---|---|
| 40 | ✅ 支持 | 最小可用尺寸 |
| 50 | ❌ 不支持 | 返回 400 |
| 60 | ❌ 不支持 | 返回 400 |
| 64 | ❌ 不支持 | 返回 400 |
| **80** | **❌ 不支持** | **原代码默认使用的尺寸！导致所有 QQ 头像请求 400！** |
| 100 | ✅ 支持 | |
| 120 | ❌ 不支持 | 返回 400 |
| 140 | ✅ 支持 | |
| 240 | ✅ 支持 | 返回图片实际为 40px，可拉伸 |
| 640 | ✅ 支持 | 最大可用尺寸 |

### 3.5 尺寸自动映射方案

为避免调用方误用尺寸，前后端均实现了 `normalizeQQSize(size)` 函数，将任意输入值映射到 **不小于该值的最近支持档位**：

```
输入尺寸 → 归一化结果
─────────────────────────
  ≤0    →  100 （默认值）
  1~40  →  40
 41~100 →  100 （原 80 → 100 ✨ 关键修复点）
101~140 →  140
141~240 →  240
241~640 →  640
  >640  →  640 （封顶）
```

## 四、Gravatar 头像获取详解

对于 QQ 邮箱以外的其他邮箱，统一使用 **Gravatar** 服务获取头像。

### 4.1 Gravatar 介绍

Gravatar（Globally Recognized Avatar）是一种全球通用的头像服务，用户在 gravatar.com 注册邮箱并上传头像后，在任何支持 Gravatar 的网站都会自动显示该头像。如果邮箱未注册，则返回一张由邮箱哈希生成的「几何图案」（identicon），确保同样的邮箱永远得到同样的默认图案。

### 4.2 URL 格式

```
https://www.gravatar.com/avatar/{邮箱MD5哈希}?s={尺寸}&d={默认图案}&r={分级}
```

| 参数 | 本项目取值 | 说明 |
|---|---|---|
| 路径部分 | `md5(trim(lowercase(email)))` | 邮箱必须先转小写、去空格，再做 MD5 |
| `s` | 调用方传入（评论区为 80） | 图片尺寸 1~2048 |
| `d` | `identicon` | 无头像时默认返回「几何图案」 |
| `r` | `g` | 分级 G = 适合所有年龄段 |

### 4.3 默认图案类型（`d` 参数可选值）

| 值 | 效果 | 本项目是否使用 |
|---|---|---|
| `404` | 无头像时返回 HTTP 404 | ❌ |
| `mp` | 神秘人（灰色卡通轮廓） | ❌ |
| **`identicon`** | **基于邮箱哈希的几何图案** | ✅ 默认 |
| `monsterid` | 小怪物 | ❌ |
| `wavatar` | 自动生成人脸 | ❌ |
| `retro` | 8bit 复古像素头像 | ❌ |
| `robohash` | 机器人 | ❌ |
| `blank` | 透明 PNG | ❌ |

## 五、兜底策略（前端 ui-avatars）

如果后端返回的头像 URL 加载失败（例如 QQ 号不存在、网络问题、域名被墙等），前端使用 **ui-avatars.com** 服务做最后一层兜底：

```
https://ui-avatars.com/api/?name={邮箱前缀}&size={尺寸}&background=random&color=fff&bold=true
```

效果：用邮箱 `@` 之前的部分生成一张彩色首字母头像，背景色随机。

如果连邮箱都为空（用户未填邮箱），则直接在页面上用 CSS 绘制一个方块，显示昵称的第一个字符。

## 九、常见问题 FAQ

**Q1：为什么我的 QQ 邮箱 `abc123@qq.com` 不显示 QQ 头像？**
A：正则只认「纯数字用户名」的 QQ 邮箱，带字母的不是合法 QQ 号。这类邮箱会走 Gravatar。

**Q2：为什么评论里头像尺寸输入了 80，但实际返回的是 100？**
A：正常行为，尺寸自动归一化到了下一档支持尺寸，以保证图片能正常加载。

**Q3：博主账号 `3181088318@qq.com` 为什么显示真实 QQ 头像？**
A：这是配置文件中写死的博主邮箱（[config.yaml L48](file:///e:/goCode/goFile/gin博客/configs/config.yaml#L48)），正好是 QQ 邮箱，经过 GetAvatarURLByEmail 后正确走到 QQ 分支并返回 qlogo.cn 真实头像。

**Q4：有些 QQ 号头像请求成功，但图片是默认企鹅？**
A：这是 qlogo.cn 对未设置头像 / 设置了非公开头像的 QQ 号的默认返回值，属正常现象，无需处理。
