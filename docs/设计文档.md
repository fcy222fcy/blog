# 博客系统前后端设计文档

## 一、项目概述

### 1.1 项目背景

本项目是一个个人博客系统，包含前台展示和后台管理两大模块。作者是一名 Go 开发者，希望通过这个博客分享 Golang 开发、AI 和 NAS 折腾经验。

### 1.2 项目目标

- **前台**：提供良好的阅读体验，支持文章浏览、分类筛选、标签过滤、评论互动
- **后台**：提供完善的内容管理能力，支持文章、分类、标签、评论、友链等管理
- **性能**：支持高并发访问，页面加载快速
- **可维护性**：代码结构清晰，易于扩展和维护

### 1.3 技术栈

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| **前端** | Vue 3 + Vite | 渐进式框架 + 极速构建工具 |
| **UI 框架** | Element Plus | 组件库（后台管理） |
| **样式** | Tailwind CSS | 原子化 CSS 框架 |
| **状态管理** | Pinia | Vue 3 官方状态管理 |
| **路由** | Vue Router 4 | 官方路由 |
| **HTTP** | Axios | 请求库 |
| **后端** | Go + Gin | 高性能 Web 框架 |
| **ORM** | GORM | Go 语言 ORM 库 |
| **数据库** | MySQL | 关系型数据库 |
| **缓存** | Redis | 热点数据缓存 |
| **认证** | JWT | 无状态认证 |
| **日志** | Zap + Lumberjack | 结构化日志 + 日志轮转 |

### 2.3 依赖注入

项目采用**构造函数依赖注入**，依赖关系：

```
Controller 依赖 Service 接口
Service    依赖 Repository 接口
Repository 依赖 *gorm.DB
```

统一在 `internal/app/app.go` 的 `initDependencies()` 里装配：

```go
// Repository
userRepo := repository.NewUserRepository(a.mysqlDB)
articleRepo := repository.NewArticleRepository(a.mysqlDB)

// Service
userSvc := service.NewUserService(userRepo)
articleSvc := service.NewArticleService(articleRepo)

// Controller
userCtrl := api.NewUserController(userSvc)
articleCtrl := api.NewArticleController(articleSvc)
```

### 3.3 前台页面路由

采用 **Hash 路由** 实现 SPA 效果：

| 路由 | 页面 | 说明 |
|------|------|------|
| `#home` | 首页 | 文章列表 + 每日一问 |
| `#archives` | 文章归档 | 按年份分组展示 |
| `#links` | 友链 | 友情链接列表 |
| `#media` | 娱乐 | 影视/游戏记录 |
| `#about` | 关于我 | 个人介绍 |
| `#post-{slug}` | 文章详情 | 文章内容 + 评论 |

**代码示例**：

```go
// user_interface.go - 接口定义
type UserService interface {
    GetUserByID(id uint) (*entity.User, error)
    GetUserByUsername(username string) (*entity.User, error)
}

// user_service.go - 实现
type userService struct {
    userRepo repository.UserRepository
}

func NewUserService(userRepo repository.UserRepository) UserService {
    return &userService{userRepo: userRepo}
}

func (s *userService) GetUserByID(id uint) (*entity.User, error) {
    return s.userRepo.FindByID(id)
}
```

### 4.4 认证设计

#### 4.4.1 JWT 流程

```
┌─────────────┐                    ┌─────────────┐
│   客户端     │                    │   服务端     │
└──────┬──────┘                    └──────┬──────┘
       │                                  │
       │  POST /auth/login                │
       │  {username, password}            │
       │ ────────────────────────────────→ │
       │                                  │
       │                                  │ 验证用户名密码
       │                                  │ 生成 JWT Token
       │                                  │
       │  {token, expires_at}             │
       │ ←──────────────────────────────── │
       │                                  │
       │  存储 Token (localStorage)        │
       │                                  │
       │  GET /articles                   │
       │  Authorization: Bearer {token}   │
       │ ────────────────────────────────→ │
       │                                  │
       │                                  │ 验证 Token
       │                                  │ 解析用户信息
       │                                  │
       │  {code: 0, data: [...]}          │
       │ ←──────────────────────────────── │
```

#### 4.4.2 Token 存储

- **前台**: `localStorage` 存储
- **后台**: `localStorage` 存储
- **请求头**: `Authorization: Bearer {token}`
